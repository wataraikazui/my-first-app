スマホアプリの開発と公開

開発中のアプリ
📸
撮ったヨ
職人が主体 ／ 各社へ送る
先行公開を準備中

現場で写真を撮り、物件名と内容を選ぶだけで、件名・ファイル名が自動でつくられ、決めた宛先へメールで送れます。 職人さんが、取引先ごとに送り分けられる少し高機能な方。まずこちらをMVPとして先に公開します。

  • 撮る → 物件名・内容を選ぶ → 件名／ファイル名が自動生成
  • 宛先を選んで、端末のメールでそのまま送信
  • 写真は自動で縮小（長辺2048px）
  • データ収集なし（端末内で完結）
📨
撮ってネ
会社が職人に配って使ってもらう
次のバージョンで

「撮ったヨ」の双子。宛先を自社だけに固定し、操作を最小にしたもの。会社が職人さん・協力業者さんに配って使ってもらう道具です。 物件名のリストも会社側で用意できる形を目指しています。

  • 宛先は自社のみ固定 ／ 操作は最小限
  • 物件名リストを会社側で用意（公開シート連携の軽い仕組み）
  • 業者ごとのコードで、自分の物件だけを表示
  • 「撮ったヨ」の派生として開発（宛先ロック＋リスト取得を追加）
開発ロードマップ
  1. ✓
    仕様を固める完了
    「撮る→選ぶ→メール送信」の流れと画面を確定。撮ったヨ／撮ってネの役割分担も整理。
  2. ✓
    1コードで両OS対応へ（Flutter）完了
    iOSとAndroidを1つのコードで作れるFlutterを本線に。Windows上でコンパイル・動作確認できる状態まで到達。
  3. ●
    実機で本物のカメラ・メールを確認進行中
    Android実機・エミュレータで、撮影とメール送信を本番同等に確認する段階。
  4. ○
    App Store へ提出（iOS）これから
    MacでiOS版をビルドし、Apple Developer 登録・スクリーンショット・審査を経て公開する段階。公開準備の書類一式（ストア文言・プライバシー・手順）は作成済み。
  5. ○
    撮ってネ（v2）と Android 展開これから
    撮ったヨの公開後に、宛先固定の「撮ってネ」と Android 版を順に。職人さんに課金の壁を立てず、会社が配る形を想定。
公開までの残り作業
  • Apple Developer Program への登録
  • Mac で iOS 版をビルドし、実機でカメラ・メールを確認
  • iPhone 専用にするか iPad も対応するかを決める
  • App Store 用のスクリーンショットを撮影
  • プライバシーポリシーを公開し、URL（サポートURL）を用意
  • App Store Connect に登録して審査へ提出
